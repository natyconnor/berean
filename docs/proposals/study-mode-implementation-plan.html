Study Mode & Verse Memory

Rollout order Queue first, then dock. Persist → Today queue → Dock → Dashboard. The dock's due badge must point at a real working review flow on day one.
How verses enter memory Hearts auto-seed. Every hearted verse (savedVerses) gets a verseMemory record. A per-verse suspended state covers "saved but not memorizing." One gesture.
Dock scope Two segments: "Notes" and "Study". "Notes" = the existing passage reading/note-writing workspace (NOT "Read" — the primary mode is note-focused, not generic reading). Search stays in the top toolbar.

Charts Hand-rolled inline SVG/CSS on existing --chart-1..5 theme tokens. No charting library.
Toolbar icon The existing BookOpen Study icon stays as a permanent fallback; the dock does not replace it.

Shared architecture

Data model & scheduler (referenced by every PR)

Two new Convex tables

A per-verse state table answers "what's due?"; an append-only log answers everything else (charts, history, future scheduler upgrades). Nothing about existing tables changes shape. 

// convex/schema.ts — additions
verseMemory: defineTable({
  userId: v.id("users"),
  verseRefId: v.id("verseRefs"),
  status: v.union(
    v.literal("new"), v.literal("learning"),
    v.literal("reviewing"), v.literal("mastered"),
    v.literal("suspended")),
  learnStage: v.number(),            // 0..3 -> full|first-letters|cloze|hidden
  ease: v.number(),                  // 1.3 .. 2.8, starts 2.3
  intervalDays: v.number(),
  dueAt: v.number(),                 // drives "due today" + dock badge
  consecutiveCorrect: v.number(),
  lapses: v.number(),
  lastReviewedAt: v.optional(v.number()),
  createdAt: v.number(),
})
  .index("by_userId_dueAt", ["userId", "dueAt"])      // Today queue + badge
  .index("by_userId_status", ["userId", "status"])
  .index("by_userId_verseRefId", ["userId", "verseRefId"]),  // upsert on heart

verseMemoryReviews: defineTable({        // append-only log
  userId: v.id("users"),
  verseRefId: v.id("verseRefs"),
  verseMemoryId: v.id("verseMemory"),
  quality: v.union(v.literal("exact"), v.literal("close"), v.literal("off")), // from classifyVerseAttempt
  accuracy: v.number(),              // 0..100 from verseAttemptAccuracy
  stage: v.number(),                 // learnStage at time of attempt
  mode: v.union(v.literal("learn"), v.literal("review"), v.literal("deck")),
  durationMs: v.optional(v.number()),
  createdAt: v.number(),
}).index("by_userId_createdAt", ["userId", "createdAt"]),

The scheduler (pure function over one attempt)

Lives at src/lib/memory-scheduler.ts — no Convex imports, fully unit-testable. Explainable in one sentence: "get it right and the gap grows; miss it and it comes back tomorrow." 

// src/lib/memory-scheduler.ts
export interface MemorySchedule {
  status: "new"|"learning"|"reviewing"|"mastered"|"suspended";
  learnStage: number; ease: number; intervalDays: number;
  dueAt: number; consecutiveCorrect: number; lapses: number;
}
export interface ReviewInput {
  quality: "exact"|"close"|"off"; accuracy: number;
  mode: "learn"|"review"|"deck"; now: number;  // pass now IN (Convex rule)
}
export function scheduleNext(s: MemorySchedule, r: ReviewInput): MemorySchedule
Rules the scheduler encodes (spec for the unit tests):
Grade While learning (stages 0–3) While reviewing (hidden)
exact advance a stage; at hidden stage → graduate to reviewing, interval 1d interval × ease; ease +0.05; consecutiveCorrect++; interval ≥ 30d → mastered
close stay on stage (retry in session) interval × ease × 0.8; ease unchanged
off drop one stage (min 0) lapse: interval → 1d, ease −0.2 (floor 1.3), lapses++, back to learning
Interval fuzz (±10%) is applied to spread review load. mastered verses still resurface (maintained, not archived).

Data flow

Heart a verse → verseMemory row created (new, due now).   Finish a card → recordAttempt mutation logs to verseMemoryReviews and applies scheduleNext to the row.   dueQueue reads by_userId_dueAt ≤ now → feeds the Today queue, the dock badge, and the dashboard.

Rules for the agent

Conventions every PR must follow

Workflow

  • One PR at a time, in order. Read files before editing.
  • After code changes run pnpm run agent:check (format → lint → typecheck). A stop hook re-runs it.
  • Run pnpm test for any PR that adds/edits a *.test.ts.
  • Regenerate Convex types with pnpm run codegen after schema/function changes.
  • Never commit or push unless explicitly asked.
  • Update docs/study-mode.md in the same PR when the data/scope/activity model changes.

Convex rules (enforced by lint)

  • Every public query/mutation has args and returns validators.
  • Check auth via the project's existing getCurrentUser helper; verify ownership on every row.
  • Never call Date.now() inside a query — pass now in as an arg.
  • Query with .withIndex(...), never .filter(...). await every promise.
  • Schedule only internal.* functions (backfill migration).
  • No any; use Doc<> / Id<> generated types.

Acceptance criteria

  • pnpm run codegen and pnpm run agent:check pass.
  • Unit tests cover: new→learning graduation path; exact/close/off at each state; lapse resets interval & drops to learning; ease clamping; mastery threshold; fuzz stays within ±10%.

Phase 1 · ships silently · server API

Convex API, hearts-as-seed & backfill

Depends on: PR 1. Unblocks: PR 3, 4.

Goal

Create the server surface for verse memory, make hearting a verse auto-seed a memory record, and backfill records for every existing hearted verse. Still no user-facing UI change.

Files

  • convex/verseMemory.ts (new) — queries & mutations (all with args+returns validators, auth + ownership checks):
    • dueQueue({ now, limit? }) — reads by_userId_dueAt where dueAt ≤ now, excludes suspended. Returns rows joined to their verseRefs.
    • dueCount({ now }) — cheap count for the dock badge.
    • recordAttempt({ verseRefId, quality, accuracy, stage, mode, durationMs?, now }) — inserts a verseMemoryReviews row, loads the verseMemory row, applies scheduleNext, patches it. Returns the updated schedule.
    • getOrCreateForVerse({ verseRefId, now }) — idempotent upsert via by_userId_verseRefId.
    • setSuspended({ verseRefId, suspended }).
    • memoryStats({ now }) — counts by status (used later by dashboard/hub).
  • convex/lib/verseMemory.ts (new, optional) — thin shared helpers (keep the query/mutation wrappers thin per repo rules).
  • Heart write path (find where savedVerses rows are inserted, e.g. convex/savedVerses.ts) — after inserting a heart, call the internal seed helper to create a verseMemory row (status: "new", dueAt = now) if none exists. When a heart is removed, leave the memory row (do not delete history); optionally mark suspended.
  • convex/migrations.ts or an internalMutation — backfill one verseMemory per existing savedVerses row lacking one.

Acceptance criteria

  • Hearting a verse creates exactly one verseMemory row (idempotent; no dupes on re-heart).
  • Backfill is idempotent and safe to run twice.
  • recordAttempt both logs and reschedules atomically; dueQueue reflects the new dueAt.
  • All functions validate args + returns, check auth, use indexes (no .filter), and contain no Date.now() in queries.
  • Update docs/study-mode.md data-model section.

Phase 3 · the cue

The Mode Dock — Notes / Study

Depends on: PR 4. Unblocks: nothing (independent after this).

Goal

A single floating pill, bottom-center, two segments — Notes and Study — with a live due badge. It is the only new persistent chrome and is engineered to disappear while you work.

Files

  • src/components/layout/mode-dock.tsx (new) — the pill. "Notes" routes to the passage workspace (the current default route / last passage tab); "Study" routes to /study. Badge = dueCount({ now }) reactive query (absent when zero — never cry wolf). Optional streak/flame.
  • App shell (find the root layout that renders tab-bar.tsx, e.g. an app-shell/root route) — mount the dock fixed, bottom-center, z-index above content, zero layout cost.
  • src/components/layout/tab-bar.tsx — stop soft-hiding the Study icon behind the reveal milestone (icon becomes a permanent fallback). Keep it functional.
  • Settings surface + convex/schema.ts userSettings — add modeDock: "auto-hide" | "always" | "off" (default auto-hide) with a get/set mutation.
  • Reuse useFeatureHint + FeatureCallout + STUDY_REVEAL_AFTER_FIRST_HEART for the one-time dock reveal on first heart.

Behavior spec

  • Auto-hide: slide down after ~24px downward scroll; return on upward scroll or when scrolling stops; always hidden while a note editor is focused and in Focus mode.
  • Keyboard: add a ⌘J / Ctrl+J toggle between Notes and Study (wire it the same ad-hoc way tab-bar.tsx handles ⌘G; use formatCommandOrControlShortcut for the tooltip).
  • A11y & motion: <nav aria-label="Mode">, focusable segments, visible focus rings, badge aria-label ("7 verses due for review"); prefers-reduced-motion → fade instead of slide.
  • Escape hatch: the off setting hides the dock entirely; the toolbar icon remains.

Acceptance criteria

  • Dock switches Notes↔Study, shows a live badge that updates as verses fall due / get reviewed.
  • Auto-hide behaves per spec; reduced-motion honored; setting toggles work; toolbar icon still navigates.
  • No layout reflow from the dock; mobile thumb-reachable. agent:check passes.

Risk Mitigation (baked into the PR sequence)
