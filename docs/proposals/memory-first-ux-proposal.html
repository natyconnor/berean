Make Memory its own mode

Verse memory graduates out of the broad "Study" paradigm into a first-class Memory mode with its own review loop, a self-directed Practice intention, and its own Packs. Study slims down to note review. Each mode answers exactly one question.

Where we are

The problem this fixes (recap)

The engine (hearts → verseMemory, a pure scheduler, an append-only review log, dashboard + global queue + library) is strong. The architecture is what's tangled: one workspace named "Study" tries to do memorization, note review, and teaching at once, and offers three different ways to practice a verse under three names ("Start review" / "Review all" / "Learn"). And a studySession welds notes and verses into one object.

Both confusions you named trace to the same root: memorization has no home of its own, so it borrows Study's page, vocabulary, and "create-a-session-first" gate. This proposal gives it a home.
North stars

Design principles

1 · One mode, one question

Notes = "capture." Memory = "get verses into my head." Study = "revisit my notes." No screen is ambiguous about which mode it is.

2 · No gate before practice

Heart a verse → it's in Memory. Opening Memory always has a "what's due" answer. Packs focus you; they're never required.

3 · One engine, two intentions

Review is the app's suggested cadence (the due queue). Practice is you choosing what to work on now. Both feed the same schedule and stats — Practice is an intention, not a parallel system.

4 · Reveal modes as they're earned

The app starts as just Notes. Memory appears on the first heart; Study appears after a few notes. Nothing empty is ever shown.

The dock

A three-segment pill — Notes Memory Study — with the live due badge on Memory. On narrow phones, segments collapse to icons (badge stays). The dock is revealed progressively (Principle 4):

  • Fresh user (Notes only): no dock at all — nothing to switch to yet.
  • After first heart: Memory unlocks; the dock appears as Notes Memory and the reveal callout points at it (P6).
  • After 4 notes: Study unlocks; the dock grows its third segment with its own one-time callout.

Keyboard: ⌘J cycles Notes → Memory → Study across currently-unlocked modes; toolbar icons mirror the same gating.

2
Memory mode · the daily driver

Memory home & the Review loop Locked

Home layout (top → bottom)

  1. Due hero — "N due today" + one primary Review button (today's "Start review", relabeled). "All caught up" at zero.
  2. Dashboard — the existing KPI row + heatmap + mastery bar + accuracy trend + forecast, moved here verbatim.
  3. Packs — your packs, each with a due count + Review and Practice actions; plus New pack (P4).
  4. Library — every verse with status + next-due, moved here verbatim (minus the suspend switch — P6).

Review: the suggested cadence

Review plays the due queue — the app's answer to "what should I work on today." The engine auto-selects the learn ladder for new/learning verses and the deck for reviewing ones, so the manual "Review all / Learn" toggle is deleted. What changes is only the set:

Entry Set played (due-only)
Memory home → Review Global due queue across all verses. The default.
A pack → Review The due verses within that pack.

When you want to work off the suggested cadence — a specific verse, a whole pack, ahead of schedule — that's Practice (P3), the same engine with a different intention.

3
Memory mode · self-directed

Practice: work whatever you want Locked

Practice is for working verses on your terms — drilling a single stubborn verse, running a whole pack before a recital, or getting ahead on a light day so a future day isn't slammed with 20 due verses. It is the same engine as Review: attempts log and reschedule identically. The only difference is who chose the set — the app (Review) or you (Practice).

The Practice board

A focused surface: one big verse card, a verse rail to jump to any verse in the set (the affordance you liked from today's Learn controls, study-verse-memory-controls.tsx), a Shuffle / In-order toggle, prev/next, and a manual stage selector — Full · Letters · Blanks · Hidden — you set freely.

Why "it counts" is the right call

  • Freedom to shape your own schedule. Nailing a verse at the Hidden stage during Practice pushes its next-due out, so a diligent user genuinely flattens future pile-ups instead of being stuck with the app's exact cadence.
  • No confusing asymmetry. Because Practice affects the schedule, having it also feed streak/heatmap/accuracy is the only consistent story — "Practice counts, full stop."
  • The scheduler is stage-aware, so it stays honest. Practicing at an easy stage only nudges the rung; real progress requires recall at Hidden. You can't cheat to mastery on easy mode.
Accepted trade-off: a miss during Practice reschedules like any miss (it can lapse a verse). The same forgiving SR framing applies — a lapse just means "see it sooner," and the streak is participation-based so it never breaks on a wrong answer. We are not adding a "Practice can only help" guard; keeping it pure preserves the one-sentence mental model.

Fast-follow (not now)

Selectable drill styles — Flip card, First-letters, Reference → text / text → reference, word-bank — layer onto this same board later. Out of scope for the first cut.

4
Collections · Option C

Packs: scope or custom Locked

A Pack is a named set of verses you're memorizing, living in Memory. When creating one, you pick how it's populated:

Scope pack

Built from a Bible scope using the same picker as study sessions today (books + chapter ranges + tags). It gathers the hearted verses that match.

Live: as you heart new verses in that scope, they join automatically. Great for "everything I'm memorizing in Romans."

Custom pack

A hand-picked, ordered list. Add verses from the reader's heart menu, the Library (multi-select), or by searching a reference.

Static: exactly what you put in it, in your order. Great for "my Fighter Verses" or a sequential passage.

Creation flow

"New pack" → name it → choose Scope or Custom. Scope reuses StudyScopeBuilder (presets, book picker, tag filter, live preview counts). Custom opens an add-verses picker. Either way you land in the pack with Review / Practice actions.

Invariant: a pack only ever contains hearted (in-memory) verses. Adding a not-yet-hearted verse to a custom pack hearts it (heart = memorize, P6), so "pack membership" and "in Memory" never drift apart.

Data sketch

A packs table (userId, name, kind: "scope" | "custom", optional scope for scope packs, createdAt). Custom membership + ordering in packVerses (packId, verseRefId, order). Scope packs resolve members live from savedVerses ∩ scope — no membership rows.

5
Study mode · the quiet room

Study slimmed + the bridge Locked

What Study becomes

/study loses the dashboard, global review, library, and the Verse Memory tab (all now in Memory). What's left:

  • A list of note collections (the existing scoped sessions), created with the existing scope builder.
  • A collection is its note list (Overview), with a Teach button that launches the teach-back deck. The tab switcher is dropped — "still pretty bare bones," as intended.

The bridge to Memory

Because a note collection is scoped by book/tag, it overlaps verses worth memorizing. A "Memorize verses in this scope" action creates (or opens) a matching live scope Pack in Memory. One-directional, optional — the modes stay separate but acknowledge they touch.

6
On-ramp & words

Heart = memorize, naming & reveal Locked

Heart has one meaning

Heart on = the verse is in Memory (being memorized). Heart off = it's not. Suspend is removed — the suspended status, setSuspended, the Library suspend switch, and the drill-down suspend control all go away. Un-hearting keeps the verse's history inactive and resumable, so re-hearting picks up exactly where it left off — but the user only ever sees "in Memory / not in Memory," never a third state.

Progressive reveal & onboarding

  • Memory is hidden from the dock/toolbar until the first heart. On that heart, the dock appears with a callout: "This verse is now in Memory" pointing at the new segment + badge (repurposes the existing STUDY_REVEAL_AFTER_FIRST_HEART hint).
  • Study is hidden until 4 notes exist. On crossing that line, the Study segment appears with its own callout: "Your notes, grouped for review." (new milestone + FeatureCallout).
  • First open of Memory: a one-card explainer — "Heart verses to add them here · Review what's due each day · Make packs to focus." Replaces the old three-bullet Study welcome.
Practicalities

Data & migration implications

Area Impact
Verse memory engine (verseMemory.ts, scheduler, reviews log) Kept. Re-homed in the UI; scheduler + append-only log unchanged. Practice attempts flow through the same recordAttempt (optionally tagged mode: "practice" for analytics only — no special filtering, since Practice counts fully).

New: Packs packs (+ packVerses for custom). Scope packs reuse the session scope shape and resolve live.
Existing studySessions Become note collections as-is (they already carry a note scope). No verse data migrates into packs.
Routes Add /memory (+ /memory/pack/$id, + Practice). Keep /study (slimmed). Redirect old dashboard deep-links → /memory.
Move, don't rewrite dashboard/*, study-library.tsx, study-today-queue.tsx, verse-detail.tsx → a new memory/ folder. Delete the session verse-memory path; strip the "Review all / Learn" toggle.

Net: mostly re-homing + re-labeling, plus one new collection type (Packs) and one new surface (Practice). The risky engine work is done and stays put.
