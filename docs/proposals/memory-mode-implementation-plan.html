Memory Mode & Packs

Split the tangled "Study" workspace into a first-class Memory mode (verse memorization: Review + Practice + Packs) and a slim Study mode (note collections) — shipped as a sequence of small, independently green pull requests.

7 pull requests no new runtime deps reuses existing engine
How to use this document. Each numbered block below is one pull request. Do them in order. Do not start the next PR until the current one is green: implement only what that PR lists, run pnpm run agent:check (and pnpm test where a PR touches a *.test.ts), satisfy the PR's acceptance criteria, then stop. The Architecture and Conventions sections are shared context for every PR.
Settled decisions

Decisions locked before implementation

Reconciled with the product owner in docs/proposals/memory-first-ux-proposal.html (rev 3). Treat as fixed constraints.

Decision Resolution
Three modes Notes · Memory · Study. Memory is first-class; Study is notes-only and bare.
Navigation Three-segment Mode Dock (Notes / Memory / Study), due badge on Memory, revealed progressively.
Heart = memorize Single meaning: a verse is "in Memory" iff it is hearted (savedVerses). Un-hearting keeps its verseMemory history inactive & resumable (row untouched); re-hearting resumes it. Suspend is removed.
One review verb Review plays the due queue (auto-picks ladder/deck). The "Review all / Learn" toggle is deleted.
Practice A self-directed board (verse rail + order toggle + manual stage selector). Counts fully (D3): logs and reschedules exactly like Review — same engine, different intention.
Packs (Option C) A Pack is a named verse set that is either a scope pack (live, reuses the session scope picker) or a custom pack (static ordered list). A pack only ever contains hearted verses.
Study slimmed + bridge Sessions become note-only (Overview + Teach; no tab switcher). "Memorize verses in this scope" spins up a live scope Pack.
Progressive reveal Memory reveals on the first heart; Study reveals after 4 notes. No dock until a second mode unlocks.
Names Mode = Memory; verse set = Pack.

Shared architecture

Data model & core contracts (referenced by every PR)

Two new Convex tables (Packs)

A pack is a per-user named verse set. Scope packs store a scope (identical shape to studySessions.scope) and resolve their members live from savedVerses ∩ scope. Custom packs store explicit ordered membership rows. 

// convex/schema.ts — additions
packs: defineTable({
  userId: v.id("users"),
  name: v.string(),
  kind: v.union(v.literal("scope"), v.literal("custom")),
  scope: v.optional(v.object({          // present iff kind === "scope"; same shape as studySessions.scope
    books: v.array(v.string()),
    chapterRanges: v.optional(v.array(v.object({
      book: v.string(), startChapter: v.number(), endChapter: v.number(),
    }))),
    tags: v.array(v.string()),
    tagMatchMode: v.union(v.literal("any"), v.literal("all")),
  })),
  createdAt: v.number(),
  lastOpenedAt: v.number(),
}).index("by_userId_lastOpenedAt", ["userId", "lastOpenedAt"]),

packVerses: defineTable({       // custom-pack membership (ordered)
  userId: v.id("users"),
  packId: v.id("packs"),
  verseRefId: v.id("verseRefs"),
  order: v.number(),
  createdAt: v.number(),
})
  .index("by_userId_packId_order", ["userId", "packId", "order"])
  .index("by_packId", ["packId"])
  .index("by_userId_packId_verseRefId", ["userId", "packId", "verseRefId"]),

The "in Memory = hearted" contract

A verse is active/in-Memory exactly when a savedVerses row exists for it. The verseMemory row is durable history and may outlive the heart. Therefore every read that reports the live memory set must filter to hearted verses: dueQueue, dueCount, memoryStats, masteryDistribution, reviewForecast (and listLibrary, already hearted-aware). Un-heart therefore needs no write change; re-heart resumes automatically because the row is untouched. There is no suspended state to reconcile — it is removed in PR 1.

Data flow

Heart a verse (Notes) → savedVerses.toggle inserts + seedVerseMemory → the verse is in Memory. Open Memory → dueQueue({now}) (heart-filtered) drives the Review hero/badge; Review plays the due set through the learn ladder / deck, each card calling recordAttempt which logs + scheduleNext. Practice plays any chosen set (a pack, or picked verses) through the Practice board, recording with mode:"practice" — identical scheduling. Packs organize verses: scope packs resolve live from savedVerses ∩ scope; custom packs from packVerses. A Study collection's "Memorize verses in this scope" calls packs.create({kind:"scope", scope}).

Rules for the agent

Conventions every PR must follow

Workflow

  • One PR at a time, in order. Read files before editing.
  • After code changes run pnpm run agent:check (format → lint → typecheck). A stop hook re-runs it.
  • Run pnpm test for any PR that adds/edits a *.test.ts.
  • Run pnpm run codegen after any schema/function change (regenerates convex/_generated).
  • Never commit or push unless explicitly asked.
  • Update docs/study-mode.md in the same PR whenever the data/scope/activity model or naming changes.

Convex & TypeScript rules (enforced by lint)

  • Every public query/mutation has args and returns validators.
  • Auth via getCurrentUserId / getCurrentUserIdOrNull (convex/lib/auth.ts); verify per-row ownership. Queries safe to call unauthenticated return []/0/null.
  • Never call Date.now() in a query — pass now in (client uses useLiveNow()). Mutations may use Date.now().
  • Query with .withIndex(...), never .filter(...); await every promise; paginate unbounded lists (paginationOptsValidator).
  • Schedule only internal.* functions (migrations); no any — use generated Doc<>/Id<>.
  • Only narrow a schema union after verifying no existing row uses the removed member (a narrowing otherwise fails the push). PR 1 narrows verseMemory.status to drop suspended — sanctioned because the DB has none. Widening (e.g. mode += "practice") is always safe.
UI conventions. Tailwind 4 + shadcn primitives (src/components/ui/*), lucide icons, framer-motion (honor prefers-reduced-motion). New routes use the file-route + thin src/components/routes/*-page.tsx pattern. Reuse useLiveNow() for any now arg. Fire attempt writes fire-and-forget via useRecordVerseAttempt (never block UI).
